# שאלון חוזקות | שוברות תקרה

מיני-אפליקציה דיגיטלית לשאלון חוזקות האופי (מבוסס VIA), ממותגת ל**שוברות תקרה · לאה גרוס**.
המשתמשת עונה על 48 משפטים בסולם 1–5, והאפליקציה מחשבת אוטומטית ומציגה את **5 חוזקות האופי המובילות** שלה.

## איך זה עובד
- 48 שאלות = 24 חוזקות אופי × 2 שאלות לכל חוזקה (אחת חיובית, אחת הפוכה).
- לכל שאלה סולם אחיד: 1 = מאוד לא מתאים … 5 = מאוד מתאים.
- חישוב הציון: שאלה חיובית `= הערך`, שאלה הפוכה `= 6 − הערך`. ציון חוזקה = סכום שתי השאלות (טווח 2–10).
- מציגים את 5 החוזקות עם הציון הגבוה ביותר.

## הרצה מקומית
פותחים את `index.html` בדאבל-קליק בכל דפדפן, אין צורך בהתקנה או שרת. עובד גם אופליין.

## הקבצים
| קובץ | תפקיד |
|------|--------|
| `index.html` | מבנה הדף (פתיחה · שאלון · תוצאות · פוטר) |
| `data.js` | 48 השאלות, 24 החוזקות, הסולם וקישורי הרשתות |
| `app.js` | לוגיקה: רינדור, ולידציה, חישוב הציונים, הדפסה |
| `styles.css` | עיצוב ממותג (נייבי + זהב), RTL, סגנון הדפסה |
| `assets/` | לוגו, מטוס הזהב, ותיקיית `fonts/` לפונטים המותגיים |

## פונטים מותגיים
פונטי המותג הם **Fb Atzofen** (כותרות) ו-**Almoni ML v5 AAA** (גוף). הם אינם webfont חינמי,
ולכן ברירת המחדל היא fallback מ-Google Fonts (Heebo / Assistant).
כדי להשתמש בפונטים המקוריים: מניחים את קבצי ה-`.woff2` בתיקייה `assets/fonts/`, ומסירים את
ההערה מבלוק ה-`@font-face` בראש `styles.css`.

## שליחת תוצאות למייל (EmailJS)
האפליקציה יכולה לשלוח למשתמשת את התוצאות למייל. זה עובד דרך EmailJS (בלי שרת). הקמה חד-פעמית:

1. נרשמים בחינם ב-[emailjs.com](https://www.emailjs.com/).
2. **Email Services → Add New Service** → מחברים את הג'ימייל/דומיין ממנו יישלחו המיילים. מעתיקים את ה-**Service ID**.
3. **Email Templates → Create New Template**. בתבנית:
   - שדה **To Email**: `{{to_email}}`
   - **Subject**: למשל `החוזקות שלך · שוברות תקרה`
   - **Content** (במצב HTML): שמים `{{{results_html}}}` (שלושה סוגריים = מאפשר HTML מעוצב).
     לחלופין, לתבנית טקסט פשוטה: `{{results_text}}`.
   - שומרים ומעתיקים את ה-**Template ID**.
4. **Account → General** → מעתיקים את ה-**Public Key**.
5. פותחים את `data.js` וממלאים ב-`EMAILJS_CONFIG` את שלושת הערכים במקום `REPLACE_ME`.

עד שממלאים את הערכים, כפתור "שלחי לי למייל" יציג הודעה שהשליחה עוד לא חוברה.

## פריסה ל-Vercel (דרך GitHub)
1. יוצרים repo חדש ב-GitHub ומעלים אליו את כל הקבצים בתיקייה זו.
2. ב-[vercel.com](https://vercel.com) → **Add New → Project → Import** את ה-repo.
3. אין צורך בהגדרות: Vercel מזהה `index.html` ופורס כאתר סטטי. לוחצים **Deploy**.
4. מקבלים כתובת חיה (למשל `https://your-project.vercel.app`) לשיתוף.

> אין צורך ב-`vercel.json`, זהו אתר סטטי טהור.
